# 02. Ценность и ROI

## Цель документа

Найти и формализовать ценность продукта и **как её обосновать** на встрече с партнёром или внутри Diginetica. Документ — основа для слайдов KPI, КП и QBR.

**Когда использовать:** фаза 2–4 sales process (impact forecast → КП), внутренняя защита, QBR с партнёром.

---

## Три слоя ценности

### Слой 1 — Поисковый (измеримый)

| Метрика | Что значит | Как обосновать на встрече | Источник |
|---------|-----------|---------------------------|----------|
| **Нулевые запросы ↓** | Меньше поисков без результата | «Запрос "змея кольцо" — N раз/мес, раньше 0 SKU, после form=snake — M SKU» | Studio `zero_to_nonzero`, CH logs |
| **Fallback → exact** | Запасная выдача становится точной | Аптека: Class I 5.7% → 3.5% | Search quality eval (E/S/I/C) |
| **NDCG@20 ↑** | Лучшее ранжирование в топ-20 | Ювелирка: +~9% в среднем | Studio `ndcg_store` |
| **Precision@20 ↑** | Больше релевантных в топе | Аптека: +0.010 | Post-impl eval |
| **Class E ↑** | Доля «точно релевантных» | Аптека: +2.6 п.п. | SERP audit |
| **Lexicon gap closure** | Новые слова в индексе | Befree: 49% SKU с truly_new | `befree_pattern_study` |

**Правило презентации:** на высоком baseline ndcg delta мала (+0.003), но **всегда** показываем параллельно Class I, zero% или story-кейс.

### Слой 2 — Бизнесовый (моделируемый)

**Формула для слайда (как в Studio CTR-вкладке):**

```
Доп. выручка ≈ ΔCTR × частота_запросов × конверсия_поиска × средний_чек
```

**Пример расчёта (иллюстративный):**

| Параметр | Значение | Комментарий |
|----------|----------|-------------|
| ΔCTR | +0.5% | Консервативно после закрытия zero-сегмента |
| Частота затронутых запросов | 15 000 / мес | Из Studio `zero_to_nonzero_freq` |
| Конверсия поиска | 8% | Бенчмарк партнёра или 6–10% |
| Средний чек | 2 500 ₽ | Из CRM / аналитики |
| **Доп. выручка / мес** | **≈ 15 000 ₽** | 0.005 × 15000 × 0.08 × 2500 |

**Когда точных CTR нет** — показываем прогноз Studio (не выдаём за факт):

- `zero_to_nonzero_freq` — суммарная частота запросов, получивших результат
- `fallback_to_exact_freq` — запросы, ставшие точнее
- `product_reach_rows` — SKU с новой discoverability

**Дисклеймер для КП:** «Прогноз основан на lexicon gap и частоте запросов; post-impl NDCG подтверждает поисковый uplift; CTR-модель — оценка порядка величины.»

### Слой 3 — Стратегический (для enterprise)

| Ценность | Описание | Аргument для C-level |
|----------|----------|---------------------|
| **Новые данные** | Цвет с фото, узор, форма — не появятся в PIM | Конкурентное преимущество поиска без CAPEX в IT |
| **Переиспользование** | Атрибуты для SEO, рекомендаций, фильтров | Один batch — несколько каналов |
| **Удержание** | «Мы делаем то, что партнёр сам не может за 6–18 мес» | Retention vs churn к конкуренту |
| **A/B для новых** | Обогащённый каталог до контракта | Win rate в pre-sales |
| **Скорость time-to-market** | 2–4 недели vs кварталы PIM | Faster ROI на search MRR |

## Нарративы по кейсам (готовые формулировки)

### Аптека (vision, OCR)

> По запросу «стельки» в выдачу попадал чай для сосудов из-за бедного описания. После OCR с упаковки — корректный товар в топе. Class E +2.6 п.п., Class I −2.2 п.п. — покупатель видит то, что искал.

### Ювелирка (vision, form)

> Покупатели ищут «кольцо змея», «серьги череп» — в фиде только «изделие 585». Атрибут form закрыл 33% нулевых запросов; NDCG@20 +9% в среднем.

### Befree / Fashion (vision, pattern)

> 49% SKU получили truly_new лексику — слова с принта и OCR, которых не было в YML. 1747 строк в Dashboard без изменения PIM Befree.

### Petrovich (audit, negative)

> 42% current_attrs — шум. Сначала cleanup, потом enrichment. Показывает зрелость продукта: мы не заливаем всё подряд.

## Матрица обоснования по аудитории

| Аудитория | Главный аргument | Доказательство | Что НЕ говорить |
|-----------|------------------|----------------|-----------------|
| C-level партнёра | Деньги из поиска | CTR-модель + zero reduction | Технические детали LLM |
| Head of e-commerce | Меньше нулей, лучше конверсия | rank-list + график zero% | ndcg без контекста |
| IT партнёра | Не трогаем PIM | Dashboard API, CSV | «Мы изменим ваш фид» |
| Контент-директор | Не нагружаем команду | Batch автоматический | «Вам нужно размечать» |
| Наш sales | Upsell к MRR | 1×MRR = понятный чек | Скидки без approval |
| Наш CEO | Retention + differentiation | Портфолио + QBR deck | «Мало сделок» без pipeline |

## Формулы для калькулятора и КП

| Показатель | Формула | Пример (MRR 150k, 800 нов./нед) |
|------------|---------|----------------------------------|
| Batch (разовый) | `1 × MRR` | 150 000 ₽ |
| Recurring (мес) | `MRR × (5% \| 7.5% \| 10%)` | 11 250 ₽ (7.5%) |
| ROI пилота (оценка) | `(прогноз_доп_выручки × 12 − batch) / batch` | Зависит от CTR-модели |
| Payback batch | `batch / (прогноз_доп_выручки_мес)` | Целевой < 12 мес для enterprise |

Инструменты: [05_PRICING.md](05_PRICING.md), [tools/pricing_calculator.html](../tools/pricing_calculator.html).

## Возражения и ответы

| Возражение | Ответ | Доказательство |
|------------|-------|----------------|
| «Метрики маленькие (+0.003 ndcg)» | На большой базе даже +0.3% ndcg = тысячи лучших сессий; Class I −2.2 п.п. — сильнее для UX | Аптека SERP audit |
| «Партнёр сам добавит в фид» | Enterprise: 6–18 мес на новый атрибут; мы — 2–4 недели batch | Timeline slide |
| «Это просто ChatGPT по картинкам» | Feed-aware: phase1/2, collision check, 200k SKU, GPU queue, on-prem | [08_TECH_ARCHITECTURE.md](08_TECH_ARCHITECTURE.md) |
| «Дорого» | 1 MRR за весь каталог vs контент-команда (FTE × месяцы) | pricing_calculator |
| «Прогноз ≠ фact» | Пилот 5–10k SKU → NDCG до/после → масштаб | Фаза 8 process |
| «Зачем recurring?» | Новинки без атрибутов = снова нули; 5–10% MRR дешевле повторного batch | discount/fashion примеры |
| «А если фото плохие?» | Phase1 prefilter + QA выборка; scope согласуем до batch | image_description pipeline |
| «У нас уже есть атрибуты» | Audit (Petrovich): 42% шум — сначала cleanup | petrovich case |

## Артефакты для deck (обязательный минимум)

1. **Таблица метрик до/после** — 3 metric-cards + sparklines (Petrovich #3 style)
2. **Три insight-блока** — поиск / бизнес / стратегия (green + purple)
3. **Rank-list** — топ-5 запросов, ушедших из нулей (частота + SKU count)
4. **Один story-кейс** — эмоциональный якорь (стельки vs чай)
5. **Pricing block** — batch + recurring из калькулятора
6. **Footnote** — источник данных + период измерения

См. [07_METRICS_PLAYBOOK.md](07_METRICS_PLAYBOOK.md), [10_DECK_SPEC.md](10_DECK_SPEC.md).

## Шкала «достаточно ли ценности для сделки»

| Сигнал | Go | No-go / пилот |
|--------|-----|---------------|
| Lexicon gap (Studio) | > 20% | < 10% без других болей |
| zero_to_nonzero_freq | > 5000 / мес | < 1000 |
| MRR партнёра | ≥ 30k | < 30k без pilot budget |
| Визуальный gap | Есть фото + запросы | Только text, бедные фото |
| current_attrs noise | < 30% или готовы к cleanup | > 50% без cleanup plan |
