# 02. Ценность и ROI

## Цель документа

Найти и формализовать ценность продукта и **как её обосновать** на встрече с партнёром или внутри Diginetica.

---

## Три слоя ценности

### Слой 1 — Поисковый (измеримый)

| Метрика | Что значит | Как обосновать | Источник |
|---------|-----------|----------------|----------|
| **Нулевые запросы ↓** | Меньше поисков без результата | «Запрос "змея кольцо" — N раз/мес, раньше 0 SKU, после form=snake — M SKU» | Studio `zero_to_nonzero`, CH logs |
| **Fallback → exact** | Запасная выдача становится точной | Аптека: Class I 5.7% → 3.5% | Search quality eval (E/S/I/C) |
| **NDCG@20 ↑** | Лучшее ранжирование в топ-20 | Ювелирка: +~9% в среднем | Studio `ndcg_store` |
| **Precision@20 ↑** | Больше релевантных в топе | Аптека: +0.010 | Post-impl eval |
| **Class E ↑** | Доля «точно релевантных» | Аптека: +2.6 п.п. | SERP audit |
| **Lexicon gap closure** | Новые слова в индексе | Befree: 49% SKU с truly_new | `befree_pattern_study` |

### Слой 2 — Бизнесовый (моделируемый)

**Формула для слайда (как в Studio CTR-вкладке):**

```
Доп. выручка ≈ ΔCTR × частота_запросов × конверсия_поиска × средний_чек
```

**Когда точных CTR нет** — показываем прогноз Studio:

- `zero_to_nonzero_freq` — суммарная частота запросов, получивших результат
- `fallback_to_exact_freq` — запросы, ставшие точнее
- `product_reach_rows` — SKU с новой discoverability

**Пример нарратива (аптека):**

> По запросу «стельки» в выдачу попадал чай для сосудов из-за бедного описания. После OCR с упаковки — корректный товар. Class E вырос на 2.6 п.п., Class I снизился на 2.2 п.п.

### Слой 3 — Стратегический (для enterprise)

- **Новые данные**, которых не было и не будет в фиде — цвет с фото, узор, форма.
- **Переиспользование** — атрибуты для SEO, рекомендаций, других продуктов.
- **Удержание** — «мы делаем то, что партнёр сам не может за 6–18 месяцев».
- **A/B для новых** — быстрый uplift до полного контракта.

## Матрица обоснования по аудитории

| Аудитория | Главный аргумент | Доказательство |
|-----------|------------------|----------------|
| C-level партнёра | Деньги из поиска | CTR-модель + zero reduction |
| Head of e-commerce | Меньше нулей, лучше конверсия | rank-list запросов + график zero% |
| IT партнёра | Не трогаем PIM | Dashboard API, CSV upload |
| Наш sales | Upsell к MRR | 1×MRR batch = понятный чек |
| Наш CEO | Retention + differentiation | Портфолио + QBR-deck |

## Формулы для калькулятора и КП

| Показатель | Формула |
|------------|---------|
| Batch (разовый) | `1 × MRR` |
| Recurring (мес) | `MRR × (5% \| 7.5% \| 10%)` по новинкам/неделю |
| ROI пилота | `(прогноз_доп_выручки − batch) / batch` — оценочно |

Подробнее: [05_PRICING.md](05_PRICING.md), [tools/pricing_calculator.html](../tools/pricing_calculator.html).

## Возражения и ответы

| Возражение | Ответ |
|------------|-------|
| «Метрики маленькие (+0.003 ndcg)» | На большой базе и высоком baseline даже +0.3% ndcg = тысячи лучших сессий; Class I −2.2 п.п. — сильнее для UX |
| «Партнёр сам добавит в фид» | Enterprise: 6–18 мес на новый атрибут; мы — 2–4 недели batch |
| «Это просто ChatGPT по картинкам» | Фид-aware pipeline: phase1/2, дельта, 200k SKU, очередь GPU, без утечки данных |
| «Дорого» | 1 MRR за весь каталог vs стоимость контент-команды на те же поля |
| «Прогноз ≠ факт» | Пилот 5–10k SKU → NDCG до/после → масштаб с доказанным uplift |
| «Зачем recurring?» | Новинки без атрибутов = снова нули; 5–10% MRR дешевле повторного batch |
