# 01. Позиционирование продукта

## One-liner

**Мы обогащаем каталог партнёра атрибутами, которых у него нет в фиде — из текста и с фото — и заливаем в поиск через Dashboard без изменения PIM партнёра.**

### Варианты формулировки (30 сек / 2 мин / 5 мин)

| Аудитория | Формулировка |
|-----------|--------------|
| **CEO / C-level** | «Деньги из поиска без 6–18 месяцев IT-очереди: новые атрибуты в индекс за 2–4 недели, с измеримым uplift по zero и NDCG.» |
| **Head of e-commerce** | «Покупатель ищет «кольцо змея» или «стельки» — мы закрываем нули и fallback, не трогая ваш YML.» |
| **IT партнёра** | «Custom attributes через Dashboard API. PIM не меняем. CSV + TOTP — вы контролируете заливку.» |
| **Наш sales** | «Upsell 1×MRR batch + 5–10% recurring. Studio показывает прогноз до подписания.» |

---

## Проблема рынка (три уровня)

### 1. Покупатель ищет то, чего нет в карточке

Пользователь вводит «кольцо змея», «стельки ортопедические», «футболка леопард» — в фиде нет соответствующих полей. Поиск возвращает ноль или fallback на нерелевантные товары (чай вместо стелек).

**Симптомы в метриках:**
- Рост доли нулевых запросов (zero%) по визуальным и брендовым терминам
- Class I (irrelevant) — выдача «что-то нашлось, но не то»
- NDCG@20 ниже бенчмарка вертикали при богатом ассортименте

**Почему это бизнес-проблема:** каждый zero = потерянная сессия поиска; каждый Class I = снижение доверия к поиску и конверсии.

### 2. Enterprise не может добавить атрибут в фид

У партнёра 30–50k SKU, согласования с IT и контент-командой, очередь на изменение YML/XML — **6–18 месяцев** на новое поле. Пока ждут, конкуренты с богатым каталогом забирают трафик.

**Типичные блокеры:**
- PIM принадлежит другому департаменту
- Поставщики не отдают структурированные данные
- Любое изменение фида = регрессионное тестирование интеграций
- Нет KPI у контент-команды на «поисковую discoverability»

**Наш ответ:** обход PIM через Diginetica Dashboard custom attributes — заливка за 1–3 дня после batch.

### 3. Визуальная информация не попадает в индекс

На фото есть цвет металла, форма украшения, принт, надпись на упаковке — в фиде нет или **противоречит** тексту. Поиск по словам не видит визуальные признаки.

**Примеры gap:**
- Ювелирка: form=snake видна на фото, в title только «кольцо 585»
- Fashion: принт «leopard» на ткани, в description — «хлопок 100%»
- Аптека: торговое название на упаковке, в YML — generic «БАД»

## Наше решение: два стрима + одна оценка

| Компонент | Репозиторий | Порт | Роль |
|-----------|-------------|------|------|
| **Attr Impact Studio** | attr-impact-studio | 5050 | Диагностика, прогноз ROI, NDCG до/после, lexicon gap |
| **attributes_extraction** | attributes_extraction-main | 8501 | Текстовый стрим: извлечение из описаний/params |
| **image_description** | image_description-main | 7860 | Vision-стрим: OCR, цвет, узор, форма |

### Замкнутый цикл продукта

```
Диагностика (Studio) → Прогноз impact → Extraction (text/vision)
    → QA выборка → Dashboard upload → Индексация → Post-impl eval → QBR
```

**Ключевое отличие от «разметки ради разметки»:** каждый шаг привязан к поисковым метрикам (zero, ndcg, Class E), а не к «количеству заполненных полей».

## Позиция в линейке Diginetica

| Аспект | Позиция |
|--------|---------|
| Отношение к search-as-a-service | **Дополнительная опция**, не замена базового поиска |
| Retention enterprise | Кастом без IT-очереди партнёра — «мы делаем то, что они сами не могут» |
| Win new partners | A/B с обогащённым каталогом до полного контракта |
| Монетизация | Отдельная строка в КП: 1×MRR batch + 5–10% recurring |
| Не конкурирует с | Синонимами, ранжированием, автокомплитом — **дополняет** индекс |

### Когда предлагать продукт

- Партнёр жалуется на нули / «поиск не понимает запросы»
- Studio показывает lexicon gap > 20%
- Вертикаль с визуальным контентом (fashion, jewelry, pharma packaging)
- Retention-риск: конкурент предлагает «поиск + enrichment»

### Когда НЕ предлагать

- Чистый фид, нет gap (Studio step 3–4)
- Шумные current_attrs без cleanup (сначала audit — кейс Petrovich)
- MRR < 30k без appetite на пилот
- Партнёр хочет переписать PIM, а не обогатить поиск

## Конкурентное отличие

| Критерий | Мы | Ручная разметка | SaaS PIM enrichment | ChatGPT-обёртка |
|----------|-----|-----------------|---------------------|-----------------|
| Скорость batch | 2–4 недели | 6–18 мес | 2–3 мес | дни, но без масштаба |
| Поисковая оценка | Studio NDCG, zero, Class E | нет | редко | нет |
| Данные | On-prem / локальные модели | in-house | утечка в SaaS | API наружу |
| Масштаб | 200k SKU, GPU-очередь | линейно по людям | лимиты API | 100–500 карточек |
| Интеграция | Dashboard API, без PIM | изменение фида | изменение PIM | ручной экспорт |
| Vision pipeline | Phase1 prefilter + Phase2 | фото не смотрят | generic CV | prompt-only |

### Messaging pillars (для deck и sales)

1. **Без PIM** — заливка через Dashboard, партнёр контролирует TOTP
2. **С доказательствами** — 7 кейсов, NDCG/zero/Class E до подписания scope
3. **On-prem** — каталог и фото не уходят в публичные API
4. **Не обёртка** — feed-aware pipeline, дельта, dedupe, QA

## Что НЕ является продуктом

| Не продукт | Почему | Куда направить |
|------------|--------|----------------|
| Ручная разметка контент-командой | Не масштабируется, нет SLA | Партнёр делает сам или аутсорс |
| Переписывание PIM / master data | Другой бюджет, другой ROI | IT партнёра |
| SEO-статьи и копирайтинг | Не влияет на поисковый индекс | Маркeting партнёра |
| Замена фида | Мы **дополняем** custom attrs | — |
| Синонимы и query rules | Базовый search MRR | Search team |
| Полный SERP redesign | Out of scope | Product search |
