# Demo cases для partner defense — как делать (Zolla и дальше)

## Жёсткие правила

1. **Один кейс = максимум фильтров с фото**, не один `focus`-атрибут.
   - Плохо: пальто → только «Капюшон=да».
   - Норм: капюшон + застёжка + пояс/кулиска + длина + стёганый (если видно).

2. **Collision со старым extract / params — не gap.**
   - Если в clothing уже «полоски», а vision вернул «полоска» — в блоке «Уже знали», не в «достали новинку».
   - Title («с капюшоном») ≠ collision для facet: name не фильтр UI; помечаем `mention_in_title`, но показываем.

3. **Спрос vs «на вырост».**
   - Блок A: атрибуты с сильным CH intent (`searched_strong` + примеры запросов и volume).
   - Блок B: видимые на фото facet-кандидаты без сильного поиска (`extractable_no_facet_yet`).
   - Не врать, что «ищут кулиску», если в top-5k пусто — честно сказать «достаём как facet».

4. **Не засорять карточку.**
   - Boolean «нет» (капюшон/карманы) на футболке — не показывать.
   - «однотонный» как единственный принт — обычно шум.
   - Материал/% состава vision не выдумывать; если шифон/замша в name — это фид/title, не «достали с картинки».

5. **Воротник / вырез.**
   - `круглый` = crew / обычный круглый вырез футболки.
   - `отложной` = рубашечный / поло с отворотом.
   - Путать их = брак демо.
