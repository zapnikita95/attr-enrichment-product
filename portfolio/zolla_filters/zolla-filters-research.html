Короткий вывод. Фид Zolla хорошо закрывает карточку (цвет, размер, скидка), но не даёт фильтров визуального стиля, которые люди уже пишут в поиск: принт, рукав, длина, капюшон, застёжка, силуэт. Мы нормализуем их в фильтры с фиксированным набором значений («да/нет», а не свободный текст вроде «с капюшоном / капюшон есть»).

5. Как выбираем формат фильтра

  1. Есть спрос и значения ещё нет в фиде как нормальный фильтр?
  2. Вопрос «есть / нет» про деталь? → формат да / нет (капюшон, карманы).
  3. Есть стандартный набор взаимоисключающих вариантов? → один из списка (длина mini/midi/maxi; рукав короткий/длинный/без рукавов).
  4. Несколько значений сразу возможны? → несколько из списка (принт).
Пример: капюшон — не свободный текст, а фильтр «да / нет», потому что покупательский вопрос бинарный («куртка с капюшоном»).

  • Узор / принт: полоска — уже было в данных карточки, не считаем новым gap

Ищут в поиске → станет фильтром

  • Длина изделия: укороченный (Длина топа укороченная) → фильтр: Длина изделия = укороченныйищут в поиске Zolla · примеры: платье макси (263), юбка миди (188)
  • Длина рукава: без рукавов (На топе отсутствуют рукава) → фильтр: Длина рукава = без рукавовищут в поиске Zolla · примеры: рубашка мужская с коротким рукавом (660), джемпер женский с коротким рукавом (301)

Достаём с фото (на вырост)

  • Воротник / вырез: V-образный (У топа V-образный вырез) → фильтр: Воротник / вырез = V-образныйспрос слабее — фильтр всё равно полезен на витрине
