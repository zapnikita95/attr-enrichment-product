## Негация = rejected (как в extraction)

В `attributes_extraction` уже есть `negation_value_filter` (`без …`, `не содержит…`): в поиск такие значения **не льём**, потому что токены `сахар` / `краситель` / `консервант` матчятся позитивным запросом.

Для vision то же самое + англ. клеймы (`MSG-free`, `Non-GMO`, `No Preservatives`):

1. На этикетке **нашли** → пишем в `vision_rejected_attrs.csv`  
2. В dashboard / gold upload → **не попадает**  
3. Это не «пропуск извлечения», а осознанный reject

Поле фида `Не содержит` (137 шт.) **не цель** для vision enrichment.

## Что НЕ доставать с картинок

| Класс | Почему |
|-------|--------|
| `без X` / `не содержит` / `-free` / БЗМЖ | Ломает поиск (negation) |
| Ноты аромата / послевкусие / «ощущения» | Уже text gold; с фото выдумывается |
| КБЖУ, срок, ТУ, бренд | Фид / низкая search relevance |
| Вино: апелласьон / регион / сорт | Уже params фида |
| `product_only` без этикетки | OCR=0 |

## Фокус-категории (без смены)

**Vision имеет смысл только на packshot:**

| P | Категории | Что искать (позитив) |
|---|-----------|----------------------|
| P0 | Снеки, фасованная молочка, бакалея/специи | форма выпуска, технология на пачке, вкус-label ≠ name |
| P1 | Напитки, мороженое, сухофрукты в пачке, pet | нарезка, тип соуса/текстура с OCR, дойпак |
| skip | Свежие овощи/фрукты, мясо без этикетки, готовая еда на тарелке, витринные торты, вино | |
